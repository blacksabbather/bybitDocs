Cancel all open orders
> **Unified account covers: Spot / Linear contract / Inverse contract / Options**<br/>
> **Normal account covers: USDT perpetual / Inverse contract**

:::info
- Support cancel orders by `symbol`/`baseCoin`/`settleCoin`. If you pass multiple of these params, the system will process one of param, which priority is `symbol` > `baseCoin` > `settleCoin`.
- **NOTE**: `category`=_option_, you can cancel all option open orders without passing any of those three params. However, for other products, you must specify one of those three params.
:::

### HTTP Request
POST `/v5/order/cancel-all`

### Request Parameters
| Parameter | Required | Type | Comments|
|:----- |:-------|:-----|------ |
|[category](../enum#category) |**true** |string |Product type <ul><li>Unified account: `spot`, `linear`, `inverse`, `option`</li><li>Normal account: `linear`, `inverse`. _Please note that `category` is **not** involved with business logic. If cancel all by baseCoin, it will cancel all linear & inverse orders_</li></ul>|
|symbol |false|string |Symbol name |
|baseCoin |false|string |Base coin |
|settleCoin |false|string |Settle coin. It does not support `spot`|
|orderFilter |false |string |Valid for `spot` **only**. `Order`,`tpslOrder`. If not passed, `Order` by default|

### Response Parameters
| Parameter | Type | Comments|
|:----- |:-----|----- |
|list |array |Object |
|> orderId |string |Order ID |
|> orderLinkId |string |User customised order ID |
