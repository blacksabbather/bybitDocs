You can move positions between sub-master, master-sub, or sub-sub UIDs when necessary
> **Unified account covers: USDT perpetual / USDC contract / Spot / Option**<br/>

:::info
* The endpoint can only be called by master api key
* UIDs must be from the same master account
* The trades generated from move-position endpoint will not be displayed in the Recent Trade (Rest API & Websocket)
* There is no trading fee
* `fromUid` and `toUid` both should be Unified trading accounts, and they need to be one-way mode when moving the positions
:::


### HTTP Request
POST `/v5/position/move-positions`

### Request Parameters
| Parameter | Required | Type | Comments|
|:----- |:-------|:-----|------ |
|fromUid |**true** |string |From UID <ul><li></li><li></li><li></li></ul>|
|toUid |**true** |string |To UID <ul><li></li><li></li><li></li></ul>|
|list |**true** |array |Object|
|> [category](../enum#category) |**true** |string |Product type. `linear`, `spot`, `option` |
|> symbol |**true** |string |Symbol name|
|> price |**true** |string |Trade price <ul><li>`linear`: the price needs to be between [95% of mark price, 105% of mark price]</li><li>`spot`&`option`: the price needs to</li><li></li></ul>|
|> side |**true** |string |Symbol name|
|> qty |**true** |string |Symbol name|

### Response Parameters
| Parameter | Type | Comments|
|:----- |:-----|----- |
|category |string |Product type |
|riskId |integer |Risk limit ID |
|riskLimitValue |string |The position limit value corresponding to this risk ID |

    testnet=True,
    api_key="XXXXX",
