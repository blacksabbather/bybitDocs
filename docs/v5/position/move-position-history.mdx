You can query moved position data by master UID api key
> **Unified account covers: USDT perpetual / USDC contract / Spot / Option**<br/>

### HTTP Request
POST `/v5/position/move-history`

### Request Parameters
| Parameter | Required | Type | Comments|
|:----- |:-------|:-----|------ |
|[category](../enum#category) |false |string |Product type. `linear`, `spot`, `option` |
|list |false |array |Object|
|> [category](../enum#category) |**true** |string |Product type. `linear`, `spot`, `option` |
|> symbol |**true** |string |Symbol name|
|> price |**true** |string |Trade price <ul><li>`linear`: the price needs to be between [95% of mark price, 105% of mark price]</li><li>`spot`&`option`: the price needs to follow the price rule from [Instruments Info](../market/instrument)</li></ul>|
|> side |**true** |string |Trading side of `fromUid` <ul><li>For example, `fromUid` has a long position, when side=`Sell`, then once executed, the position of `fromUid` will be reduced or open a short position depending on `qty` input</li></ul>|
|> qty |**true** |string |Executed qty <ul><li>The value must satisfy the qty rule from [Instruments Info](../market/instrument)</li></ul>|

### Response Parameters
| Parameter | Type | Comments|
|:----- |:-----|----- |
|result |map |Result object |
|> blockTradeId |string |Blocktrade ID|
|> status |string |Statue. `Processing`|
|> rejectParty |string |The position limit value corresponding to this risk ID |
